# Share Experiments

Steadybit offers several ways to share an experiment to make it easy to share knowledge and enable central organized rollouts.

Pick the approach that matches how much central control you need, how the design should evolve and how tightly the shared experiments should stay in sync:

* Deep link — Just sharing a useful experiment's scenario or run for shared analysis 
* [Duplicate Experiment](duplicate-experiment/README.md) — One-off, fully editable copy in another team
* [File Import/Export](file-import-export/README.md) — Move an experiment across Steadybit platforms, into version control, or CI/CD
* [Share With Teams](share-experiment/README.md) — Another team should run a specific experiment as-is, with one shared run history
* [Experiment Templates](templates/README.md) — Other teams should start from your design and adapt it to their own needs
* [Service Provided Experiment](service-provided/README.md) — The same design needs to be tested for different environments/targets, with central control over the design
